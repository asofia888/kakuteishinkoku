# 申告スナップ 📋

**個人事業主向けの確定申告(青色申告)・帳簿管理ウェブアプリ**です。
銀行口座やクレジットカードのCSV明細を読み込むと、キーワードルールで勘定科目を自動仕訳し、家事按分を計算した上で、青色申告決算書の作成に必要な科目別合計額と所得金額を自動集計します。
すべての取引から**複式簿記の仕訳帳・総勘定元帳・貸借対照表を自動作成**するため、発生主義で記帳すれば**青色申告特別控除55万円(e-Tax等なら65万円)**の帳簿要件に対応できます。消費税(インボイス)の集計・納付額試算、**適格請求書の発行(印刷/PDF)と売掛金計上**にも対応。

データはすべて端末のブラウザ(localStorage)にのみ保存され、外部送信は一切ありません。

## 主な機能

### A. CSVインポートと自動仕訳(ルールベース)
- 銀行・カード明細CSV(日付・金額・摘要)をアップロード。**Shift_JIS / UTF-8 を自動判定**し、列の並びもヘッダーの有無も自動で推測
- 日付は `2026/1/5`・`2026年1月5日` のほか **8桁数字**(`20260105`、楽天銀行等)・**和暦**(`令和7年1月5日`・`R7.1.5`、ゆうちょ等)に対応。入金・出金が別列の明細も、±符号1列(「入出金」)や「入払区分」列の明細も判定
- 摘要に含まれるキーワードを登録済みルールと照合し、勘定科目を自動割り当て(例:「Amazon」→ 消耗品費、「電力」→ 水道光熱費)。**半角カナ・全角英数の表記ゆれ**(`ﾃﾞﾝｷ`・`ＡＭＡＺＯＮ`)も NFKC 正規化で吸収して照合
- 取込前にプレビューで科目・収支種別を修正可能。重複の可能性がある行を自動検出し、**読み取れなかった行数も表示**(入金行の取りこぼし等に気づける)。列名を推測で読み取った場合は収入/支出の確認を促す
- 取込後は一覧で科目の修正・**承認**ができ、未仕訳の取引にはルールを一括再適用できる
- 私的な取引は**「対象外(プライベート)」**に割り当てると、集計・アラートから除外される

### B. 家事按分(事業割合の自動計算)
科目ごとに次のいずれかを登録できます。経費計上されない残額は**事業主貸**として扱われます。

| 方式 | 動作 | 例 |
| --- | --- | --- |
| パーセント指定 | 各取引の金額 × 事業割合(%)を経費計上 | 電気代10,000円 × 40% → 経費4,000円 / 事業主貸6,000円 |
| 固定金額指定 | 月ごとに指定額まで経費計上し、超過分は事業主貸 | 家賃80,000円・固定30,000円 → 経費30,000円 / 事業主貸50,000円 |

按分は取引の取込・編集・設定変更のたびに**全取引へ自動で一括適用**されます(按分適用済みフラグ付き)。
科目ごとに**根拠メモ**(「仕事部屋の床面積 20㎡ / 全体 50㎡」など按分割合の算定根拠)を残せます。税務調査で説明を求められたときの定石です。

### C. 確定申告用データの自動集計
- 指定年度(1/1〜12/31)の**月別売上・年間売上合計**(入出金日ベース=現金主義)
- **勘定科目別の経費年間合計**(家事按分適用後の純粋な経費額・事業主貸の内訳付き)
- **差引金額(青色申告特別控除前の所得金額)**
- 決算書用の科目別集計と取引データを**CSVエクスポート**(BOM付き・Excel対応・数式インジェクション対策済み)
- ダッシュボードにアラートを表示: 未仕訳・未承認に加え、**期ズレ候補**(翌年1月に入金された売上)と**減価償却候補**(10万円以上なのに消耗品費のままの取引。少額減価償却資産の特例の案内付き)を自動検出

### D. 複式簿記・貸借対照表(55万/65万円控除対応)
各取引に**決済手段**(普通預金/現金/クレジットカード(未払金)/売掛金/買掛金/事業主)を持たせ、全取引から複式仕訳を自動導出します。

- **仕訳帳・総勘定元帳**(残高・相手勘定付き)を自動作成、CSV出力
- **貸借対照表**(青色申告決算書の様式に合わせた期首/期末2列)。期首残高を登録すると元入金は自動算出され、**前年末からの繰越も1クリック**
- **発生主義の記帳**: 請求時に「売掛金(発生記録)」で売上計上 → 入金時は「売掛金の回収」(損益に二重計上されない)
- **カード明細の正しい処理**: カード購入は未払金として計上され、銀行側の引落し行を「カード代金の引落し」にすると自動で相殺(経費の二重計上を防ぐ)
- 家事按分の家事分・対象外の取引は**事業主貸/事業主借**として自動仕訳
- 貸借一致の検算バッジ付き。「青色申告特別控除前の所得金額」はダッシュボードの差引金額と必ず一致
- **資金移動**(ATMでの引き出し・預け入れ)は科目「資金移動」で記帳でき、現金・預金残高が正しく動く(損益に影響しない)

### E. 減価償却(自動計算)・固定資産台帳・棚卸資産
- **固定資産台帳**: 10万円以上の備品・車両などを登録すると、**耐用年数から償却費を自動計算**
  - **定額法**(償却率表と同じ率・取得年は月割り・帳簿価額1円の備忘価額まで)
  - **一括償却資産**(10〜20万円未満・3年均等)/**少額減価償却資産の特例**(青色・30万円未満・全額。**年合計300万円の上限チェック付き**)
  - 償却資産税(固定資産税)の申告対象の目安表示(150万円ライン・一括償却は対象外の案内)
  - 事業専用割合に対応(家事分は事業主貸)。除却日を入れると償却が止まる
  - 資産ごとの**償却予定表**と、青色申告決算書**「減価償却費の計算」欄のCSV出力**
- 購入の支払い行は科目を**「固定資産の取得(振替)」**にすると資産計上され、経費との二重計上を防止。台帳と取引の金額が食い違うと**照合アラート**が出る
- 償却費は毎年12/31付の**決算整理仕訳として帳簿・損益・貸借対照表に自動反映**(データ入力不要)
- **棚卸資産**: 年末の棚卸高を入れるだけで**売上原価(期首 + 仕入 − 期末)**を自動計算し、貸借対照表の「棚卸資産」にも反映
- **開業費(繰延資産)**: 任意償却に対応。登録すると開業日付で「(借)繰延資産 / (貸)事業主借」を自動起票し、好きな年に好きな額を償却できる(利益の出た年に多く償却する定石が使える)

### F. 所得税シミュレーション・証憑保存・月次推移
- **所得税シミュレーション**: 所得控除(社会保険料・iDeCo・生命保険料・医療費・ふるさと納税・配偶者/扶養など)を入力すると、
  速算表・復興特別所得税(2.1%)・**基礎控除(令和7年度改正対応: 2025年分以降58万円+時限上乗せを年分で自動切替)**・医療費/寄附金の足切りまで自動計算し、
  **納付見込み(源泉徴収差引後)**を表示。請求書の源泉徴収税額はワンクリックで集計。住民税・個人事業税の概算付き
- **証憑(領収書・請求書PDF)の添付**: 取引ごとに画像・PDFを添付(ブラウザ内のIndexedDBに保存)。
  📎バッジで一覧表示、電子帳簿保存法の検索要件(日付・金額・取引先)は取引一覧の検索で対応
- **月次推移**: ダッシュボードに月別の売上・経費・損益の推移表(決算整理は12月に計上)

### G. 消費税(インボイス)の集計・試算
- 取引ごとの**税区分**(課税10%/軽減8%/非課税/不課税)。科目から自動判定し、取引一覧で個別変更可
- **適格請求書(インボイス)なし**の仕入は経過措置(2026/9まで80%・2029/9まで50%)を適用して控除額を計算
- **本則課税・簡易課税(事業区分1〜6)・2割特例**の納付見込額を並べて比較
- 課税仕入は**家事按分後の事業分のみ**を対象にする(按分と連動)
- 免税事業者は参考表示。課税売上1,000万円超えの警告付き(税込経理・概算)

### H. 請求書発行(インボイス対応)
- 明細行(品目・数量・単価・税率10%/軽減8%/対象外)を入力して**適格請求書の要件を満たす請求書**を作成。消費税の端数処理は**1請求書あたり税率ごとに1回**(インボイス制度の要件)
- 請求元情報(氏名・**登録番号 T+13桁**・住所・振込先)を登録すると自動印字。税込/税抜単価の両対応
- **源泉徴収**(10.21%・100万円超部分は20.42%)の差引に対応
- ブラウザの印刷ダイアログから**印刷・PDF保存**(請求書だけがきれいに印刷されるレイアウト)
- **「売掛計上」ボタンで発生主義の売上計上**(借方: 売掛金 / 貸方: 売上。税率ごとに取引を分けて消費税集計とも連動)。源泉分は事業主貸として自動差引され、売掛金残高が入金予定額と一致。入金時は入金行を「売掛金の回収」にするだけ
- 複製機能で毎月の定期請求もかんたん。請求書番号は年ごとに自動採番(2026-001, 002, …)

### I. データ保全(バックアップ/復元)
- 全データ(取引・ルール・按分設定)を**JSONファイルとしてダウンロード**し、いつでも**復元**できる
- 復元時・起動時のデータは形式を検証し、壊れた要素は捨ててアプリが落ちないようにする
- localStorage への保存に失敗した場合(容量超過等)は画面上部に**バナーで警告**(無音でデータが消えない)
- **複数タブで開いても安全**: 他のタブでの変更は storage イベントで自動反映され、後から保存したタブによる上書き消失を防ぐ
- 取引を削除しても、直後に**「元に戻す」**で復元できる

## 画面構成

サイドバー付きの管理画面UI:

1. **ダッシュボード** ── 年度別の集計(損益計算書相当)・月次推移(売上/経費/損益)・経費内訳・アラート・CSV出力・バックアップ(データ管理)
2. **取引一覧(CSV読込)** ── CSV取込プレビュー、検索/フィルタ、科目・決済手段・税区分の修正、承認、対象外指定、証憑の添付(📎)、手入力追加、削除の「元に戻す」
3. **請求書発行** ── 請求元情報の登録、明細入力、適格請求書のプレビュー/印刷(PDF)、売掛金としての売上計上、複製
4. **帳簿・決算書(複式簿記)** ── 期首残高の登録(前年から自動繰越)、棚卸高の入力、貸借対照表、仕訳帳、総勘定元帳、CSV出力
5. **固定資産台帳(減価償却)** ── 資産・開業費の登録、償却費の自動計算、償却予定表、取得取引との照合、決算書「減価償却費の計算」CSV
6. **消費税(インボイス)** ── 課税事業者/計算方式の設定、税区分別の年間集計、納付額の3方式比較
7. **所得税シミュレーション** ── 所得控除の入力、納付/還付見込み、住民税・事業税の概算
8. **自動仕訳ルール設定** ── キーワード→科目ルールのCRUD(振替科目にも対応)、優先順位の並べ替え、お試し判定
9. **家事按分設定** ── パーセント/固定額の登録、根拠メモ、年度別の按分効果の試算

## データスキーマ

| ストア | フィールド |
| --- | --- |
| Transactions | id, date, amount, description, account(勘定科目。null=未仕訳 / excluded=対象外 / 振替科目), type(収入/支出), fund(決済手段: bank/cash/card/receivable/payable/owner), anbunApplied, businessAmount(按分後計上額), approved, source, createdAt, taxCategory(税区分・省略時は科目から自動判定), qualifiedInvoice(適格請求書の有無) |
| Rules | id, keyword, account(並び順=優先順位) |
| AnbunSettings | id, account, type(percent/fixed), value, memo(按分割合の根拠メモ) |
| OpeningBalances | year, cash, bank, receivable, card, payable(期首残高。元入金は自動算出) |
| TaxSettings | taxable(課税事業者か), method(本則/簡易/2割特例), simplifiedType(事業区分1〜6) |
| Invoices | id, number, issueDate, dueDate, client, clientSuffix, title, items(明細: 内容/数量/単価/税率), taxIncluded, withholding(源泉徴収), notes, linkedTxIds(売掛計上した取引), createdAt |
| Issuer | name, invoiceRegNumber(T+13桁), address, tel, email, bankInfo(請求元情報) |
| Assets | id, name, acquiredDate, cost, method(定額法/一括償却/少額特例/任意償却), usefulLife, businessRatio(事業専用割合), memo, disposedDate(除却日), deferredDep(任意償却の履歴), createdAt |
| Inventories | year, amount(年末棚卸高。年ごとに1件) |
| Deductions | year, 社会保険料・iDeCo・生命保険料・地震保険料・医療費・寄附金・配偶者・扶養・その他, blueDeduction(青色控除の選択), withholding(源泉徴収税額) |
| 証憑ファイル | IndexedDB(shinkoku-snap-files)に取引IDと紐づけて保存。バックアップJSONには含まれない |

## 使い方

```bash
npm install
npm run dev        # 開発サーバー(http://localhost:3000)
npm test           # ユニットテスト(Vitest)
npm run typecheck  # 型チェック
npm run lint       # ESLint
npm run build      # 本番ビルド(静的エクスポート: out/)
```

1. **自動仕訳ルール設定**でキーワードを確認(初回はおすすめルールが登録済み)
2. **取引一覧**で銀行・カードのCSVを取り込む(カード明細は決済手段「クレジットカード(未払金)」のまま取込。銀行側のカード引落し行は科目を「カード代金の引落し」に)
3. 未仕訳の取引に科目を割り当てて**承認**(私的な取引は「対象外」に)
4. **家事按分設定**で家賃・光熱費などの按分方法と根拠メモを登録(全取引へ自動適用)
5. **請求書発行**で請求書を作成・印刷し、「売掛計上」で発生主義の売上計上(入金前の12月請求も正しく当年の売上に)。入金があったら入金行を「売掛金の回収」に
6. 10万円以上の備品等は科目を「固定資産の取得(振替)」にして**固定資産台帳**に登録(償却費は自動計算)
7. **帳簿・決算書**で期首残高を登録(仕入がある事業は棚卸高も入力)し、貸借対照表・仕訳帳・総勘定元帳を確認/出力
8. 課税事業者は**消費税**ページで方式を設定し、納付見込みを確認
8. **ダッシュボード**で年間集計を確認し、決算書用CSVをダウンロード
9. ダッシュボードの「データ管理」から**バックアップJSONを定期的にダウンロード**(端末故障・ブラウザのデータ消去に備える)

はじめての方はダッシュボードの「サンプルデータを読み込む」でひと通りの動きを体験できます。

## 技術構成

- Next.js 15 (App Router) + React 19 + TypeScript
- Tailwind CSS v4
- データ永続化: localStorage(静的エクスポート対応・サーバー不要)
- 依存ライブラリ最小(グラフ・CSV処理は自前実装)
- テスト: Vitest(CSV解析・家事按分・年間集計・自動仕訳・バックアップ検証・複式仕訳/貸借対照表・消費税・請求書計算・減価償却)
- Lint: ESLint(eslint-config-next)
- CI: GitHub Actions(push時に lint・型チェック・テスト・ビルドを自動検証。GitHub 利用時のみ)

## 免責事項

本アプリは帳簿作成の補助を目的としたツールです。複式簿記の仕訳帳・総勘定元帳・貸借対照表を自動作成しますが、55万/65万円控除には**すべての事業取引をもれなく記帳し、売掛金・買掛金による発生主義の処理を行うこと**が前提になります(65万円はさらに e-Tax による申告または電子帳簿保存が必要)。減価償却は定額法・一括償却・少額特例・繰延資産の任意償却(平成19年4月以後取得)に対応していますが、定率法・資本的支出、事業用資産の売却損益/除却損の自動計上(売却は譲渡所得になる場合があります)には対応していません。消費税の集計は税込経理を前提とした概算、所得税シミュレーションは事業所得のみを前提とした概算です(給与所得・税額控除は未対応)。証憑の添付は電子帳簿保存法のタイムスタンプ要件を満たすものではないため、訂正削除防止の事務処理規程の整備と元データの保管を併用してください。実際の申告にあたっては、国税庁の確定申告書等作成コーナー・e-Tax の案内および税理士の確認に従ってください。
